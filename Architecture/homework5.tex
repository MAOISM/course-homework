\begin{enumerate}
    \item  宏定义中\&、！和\%操作符的作用分别是什么？
        \begin{itemize}
        \item \& 替换操作符，将宏的内部对宏参数的引用替换为调用时的实际值。
        \item ! 特殊字符操作符，强制预处理器将预定义的操作符作为原始字符传递。
        \item \% 展开操作符，用于展开文本宏或把常量表达式转换为文本表示式。
        \end{itemize}
    \item IFIDN伪指令的用途是什么?\\
        比较两个符号是否相等，但是要区分大小写，若相等则返回 True. 当要确保调用宏时传递的参数不和宏内使用的寄存器冲突时，
    \item  写出生成100到A0000000h之间的斐波那契数作为一系列编译时期的常量。
        \begin{lstlisting}[language={[ANSI]C}]
        . data
            num1 = 55
            num2 = 89
            DWORD num1
            DWORD num2
            sum = num1 + num2

            WHILE sum LT 0A0000000h
                DWORD sum
                num1 = num2
                num2 = sum
                sum = num1 + num2
            ENDM\end{lstlisting}
    \item  哪个Win32函数返回标准输入的句柄？\\
            GetStdHandle 
    \item  说出至少三个由WinMain过程所做的工作。
            \begin{itemize}
            \item 获取当前程序的句柄
            \item 装载程序使用的图标和鼠标光标
            \item 注册主窗口使用的窗口，并且定义用来接收窗口事件消息的子程序
            \item 创建主窗口
            \item 显示并更新主窗口
            \item 开始一个消息循环来接收和分派处理消息
            \end{itemize}
    \item 解释 HeapAlloc 函数的用途。\\
           它用来在指定的堆上分配内存，并且分配后的内存不可移动。调用成功则通过 EAX 返回内存块的地址，否则 EAX 为 0
    \item 给出一个调用HeapCreate函数的例子。\\
            INVOKE HeapCreate, 0, 20000, 50000
    \item 段选择子是32位长吗？\\
        不对，是16位
    \item 什么是“分段”？\\
        将多个内存段相互隔离的办法，这使多个程序可以相互隔离地运行，而不会相互干扰
    \item 代码题目
    在教材第11.2.6节的程序清单的基础，写出一个WinApp.asm程序，要求：
    \begin{itemize}
    \item[a.]实现第11.2.6节的程序清单的所有功能；
    \item[b.]当用户在主窗口的任何地方按下键盘时，程序显示消息框的标题为“Welcome”；
    \item[c.]当用户按下主窗口右上角的关闭按钮时，窗口关闭前显示消息框的文本为“Bye”。
    \item[d.]提示：可能需要在GraphWin.inc中添加  WM\_KEYDOWN          =  0100h
    \end{itemize}


\end{enumerate}
